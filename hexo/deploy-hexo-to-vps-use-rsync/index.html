利用文件同步(ssh)发布hexo到云端vps

2016-02-02
hexo

都是百度搜索惹的祸，辛辛苦苦写了blog，却发现百度不能收录github:pages，本教程来帮助大家，让百度收录我们的博客

BiBiHub利用文件同步(ssh)发布hexo到云端vps

这个世界肿么了

实在想不通自己的网站为啥在百度搜索不到，去google搜索，去百度站长提交，一直抓取失败，现在用curl模拟一下百度爬虫

结果…

看来万能的百度，不灵了,好吧，反正我有一个VPS，现在只用它来访问google，让我们把Hexo部署上去吧

首先，我们要用hexo-deployer-rsync这个插件

安装方法很简单

完整配置_config.yml，那些有默认值的可以不配

  • host: 云端vps的host
  • user: 在vps上的用户名(要保证该用户有文件夹写权限)
  • root: vps上，要同步到的路径
  • port: 端口号，一般是22端口
  • delete: 是否删除vps上的过期文件
  • args: rsync 参数
  • verbose: Display verbose messages
  • ignore_errors: Ignore errors

在VPS上安装nginx

我的VPS是centos7，安装nginx很简单，用yum命令就好了

用root启动起来验证一下(服务器上的nginx一般默认是80端口，80端口需要root权限)

访问http://remote-ip，看到这个页面，说明nginx安装成功了

这里面有两个有用的信息，第一个是/usr/share/nginx/html是默认的应用路径。第二个是/etc/nginx/nginx.conf是默认的配置文件，你可以通过配置文件，来修改端口，错误页面，应用路径等，我们暂时不去理会他。一会，我们用hexo-deployer-rsync，来把我们的hexo项目发布到/usr/share/nginx/html下

开始部署

用最简单的配置，编辑本地_config.yml文件

执行hexo发布命令

命令出现交互窗口，输入密码

发布完成后，验证一下，打开网页访问http://remote-ip

域名解析

如果你已经有域名了，且已经把域名解析到了github page上，那么就忽略这个章节把，如果你还没有域名，或者打算购买一个，那就继续往下看吧
一般来说，在你买的域名网站，会有类似解析设置的面板，一般我们要添加两条A纪录，一个主机纪录为@、一个为www，截图上的说明很清楚了，我就不啰嗦了

验证百度是否能收录网站

登录百度站长，看一看效果，点击抓取按钮，重新抓取一次网站

还是抓取失败!!!

查看错误原因，发现网站IP还是github的，点击反馈按钮，页面会提示已反馈，预计几分钟内完成更新
